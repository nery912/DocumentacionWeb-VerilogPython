Evolución de Herramientas

Gestión de Proyectos Modernos en Python

La herramienta uv es un gestor de paquetes y proyectos de alta velocidad escrito en Rust. Integra en una sola solución todo lo que antes hacías de forma fragmentada.

⚡ La Revolución Todo en Uno: PIP vs UV

Antes necesitabas un ecosistema roto de 6 herramientas separadas. Ahora usás una sola.

Característica El Flujo Viejo (PIP / Otros) El Flujo Moderno (UV)
Velocidad Instalación lenta basada en Python puro. ⚡ Ultra veloz (Escrito en Rust).
Versiones de Python Requiere herramientas como pyenv. Descarga y gestiona entornos de Python de forma nativa.
Entornos Virtuales Creación manual con virtualenv o venv. Gestión e inicialización 100% automática.
Bloqueo de Entorno Manual mediante archivos requirements.txt. Automatizado mediante uv.lock transitable.
Distribución Requiere poetry, flit o twine para publicar. Compila y publica paquetes directo a PyPI.
🛠️ Características Clave de UV ↓
🚀 Instalación ultra rápida: Ideal para árboles grandes de dependencias.
📁 Inicialización de proyectos: Genera la estructura base y el repositorio Git de un tiro.
🔒 Gestión Avanzada: Rastrea dependencias directas y transitivas automáticamente.
🧪 Soporte Dev: Instala y corre herramientas satélites en aislamiento (como Ruff, Black o pytest).
🚀 Creación de un Proyecto Python con UV ↓

Navegá hasta tu directorio de trabajo y ejecutá el comando inicializador:

uv init ejemplo

La Estructura Generada Automáticamente:

ejemplo/
├── .git/               # Repositorio Git inicializado automáticamente
├── .gitignore          # Configurado con exclusiones estándar de Python
├── .python-version     # Define qué versión exacta de Python usará el proyecto
├── README.md           # Archivo de documentación vacío
├── main.py             # Script marcador de posición
└── pyproject.toml      # Configuración del proyecto y sus dependencias
📄 main.py 
def main():\n    print("Hello from ejemplo!")\n\nif __name__ == "__main__":\n    main()
📄 pyproject.toml 
[project]\nname = "ejemplo"\nversion = "0.1.0"\nrequires-python = ">=3.13"\ndependencies = []
🔄 Migración de Proyectos Existentes y requirements.txt ↓
Caso A: El proyecto ya existe pero no usa UV

Entrá a la carpeta y ejecutá el comando base. Creará la infraestructura sin tocar tu main.py ni tu repositorio Git previo (no funcionará si ya tenés un pyproject.toml activo):

uv init
Caso B: Migrar desde un requirements.txt viejo

Importá instantáneamente todas tus librerías previas directo al control de uv:

uv add -r requirements.txt

🔹 Dependencias Directas: Paquetes declarados explícitamente (ej. Tu app requiere requests).

🔹 Dependencias Transitivas: Librerías secundarias que dan soporte a las directas (ej. requests necesita internamente de urllib3). Uv las maneja y aísla solas dentro del ecosistema.

🔒 Bloqueo y Sincronización Automática (Garantía de Entorno) ↓

Uv remueve el error clásico de "en mi máquina andaba pero en la tuya no" usando dos procesos automatizados:

  • El Bloqueo (uv.lock): Captura las versiones exactas, la arquitectura del sistema operativo y las distribuciones de Python utilizadas en tu entorno de desarrollo para asegurar una reproducibilidad matemática exacta en cualquier otra computadora.
  • La Sincronización: Instala y purga paquetes en caliente. Al ejecutar comandos mediante uv run, el sistema audita, bloquea y sincroniza automáticamente las librerías antes de arrancar, manteniendo tu sandbox impecable sin intervenciones manuales.
